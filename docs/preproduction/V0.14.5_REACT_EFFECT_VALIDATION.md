# v0.14.5 - React effect validation

## Scope

This patch resolves the three `react-hooks/set-state-in-effect` errors reported by the local ESLint validation after v0.14.4.

## Changes

- Suggestion administration now fetches data directly from the effect and only updates state from asynchronous promise callbacks.
- The personal suggestion history uses the same cancel-safe loading pattern.
- Notification hash navigation schedules expansion and scrolling inside `requestAnimationFrame` callbacks.

## Behaviour preserved

- Initial loading indicators remain visible until each request finishes.
- Unmounted screens ignore late responses.
- Notification groups still open automatically when reached from a search result or direct hash URL.
- No API, database, route, permission, or search-index changes are included.

## Validation

Run locally:

```powershell
npm run lint; if ($LASTEXITCODE -eq 0) { npx tsc --noEmit }; if ($LASTEXITCODE -eq 0) { npm run build }
```

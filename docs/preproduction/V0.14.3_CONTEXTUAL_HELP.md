# v0.14.3 - Contextual help and stable settings search

Date: 2026-07-24

## Scope

- Keep the floating Settings search dialog at a stable height while the query, results, or mobile keyboard change.
- Update Help and basic concepts with recent application features.
- Adapt Help content to the active season configuration.
- Adapt the league invitation rules summary to the target season configuration.

## Search dialog

- The dialog uses a fixed responsive height based on the small viewport unit.
- The header and input remain fixed inside the dialog.
- Only the result area scrolls.
- Result count, empty states, recent entries, and suggestions no longer resize or move the dialog.

## Contextual help

Help now presents:

- the active roster model and player capacity;
- balanced or manual calendar mode;
- single, double, or extended schedule;
- round deadlines when configured;
- mandatory or flexible sets;
- result confirmation mode;
- automatic or voting MVP only when enabled;
- season fee only when enabled;
- player incidents only when enabled;
- substitutes and replacements only when enabled;
- current application tools such as availability, scheduling, payments, notifications, activity, spectators, and suggestions.

The previous detailed sections remain available, but registration, substitutes, and MVP explanations are hidden when they do not apply.

## Invitation rules acceptance

The pre-join rules screen now derives its summary from the target season settings. It omits disabled features and describes the actual roster, calendar, scoring, payment, confirmation, MVP, incident, and substitution rules.

## Data and infrastructure

- No API contract changes.
- No database migration.
- No permission or RLS changes.
- No Settings search-index changes.
